# КАР-точки 🐦‍⬛

Минималистичное веб-приложение для запоминания слов, терминов и цитат с интервальным повторением. Работает на компьютере и телефоне, устанавливается на домашний экран как обычное приложение.

Название — каламбур: **КАР**-точки = ворона + карточки. Отсюда и иконка.

## Что умеет

- Карточки: лицо (слово/термин) + на обороте «Определение» (жирным, по центру) и необязательное «Описание» (мельче, по ширине карточки) — плюс картинка на любой стороне (с телефона или компьютера, перетаскиванием или выбором файла).
- В редакторе текст можно **выделить жирным** и добавить гиперссылку.
- При повторении карточку можно переворачивать туда-обратно сколько угодно раз (клик, тап или пробел/Enter) — удобно сверять лицо и оборот перед оценкой. Размер карточки подстраивается под текст и картинку, а если места всё равно не хватает — появляется прокрутка внутри карточки.
- Папки по темам, с цветами.
- Два алгоритма интервального повторения на выбор в настройках:
  - **SM-2** (как в Anki) — оценки «снова / трудно / хорошо / легко», интервал растёт индивидуально;
  - **Коробки Лейтнера** — 5 коробок, «помню / не помню», интервалы коробок настраиваются.
  - Прогресс каждого алгоритма хранится отдельно — переключение ничего не ломает.
- Направление показа: лицо → оборот, оборот → лицо или вперемешку.
- Лимит новых карточек в день, экспорт/импорт в JSON (резервная копия и перенос данных).
- **Массовое добавление** карточек списком (`слово — перевод`) и **автоперевод** RU↔EN в редакторе (MyMemory, нужен интернет).
- **Озвучка** на повторении: кнопка, авто при перевороте, скорость 0.5×–2×; язык по алфавиту (кириллица/латиница).
- Тёмная тема — автоматически по настройке системы.
- Два режима работы:
  - **Демо-режим** — без регистрации, данные хранятся в браузере (IndexedDB). Работает сразу, синхронизации между устройствами нет.
  - **Облако (Supabase)** — вход по почте и паролю, карточки и картинки синхронизируются между всеми устройствами. У каждого пользователя своя коллекция.

Приложение написано на чистом HTML/CSS/JS — без сборки и зависимостей. Что лежит в папке, то и деплоится.

## Быстрый старт (попробовать прямо сейчас)

Откройте `index.html` в браузере двойным кликом и нажмите «Попробовать без регистрации». Всё, кроме установки на телефон, уже работает.

## Публикация в интернете (бесплатно)

Чтобы заходить с телефона и дать доступ друзьям, выложите папку на любой бесплатный статический хостинг. Самый простой путь:

**Netlify Drop** (проще всего, ~1 минута)
1. Зайдите на https://app.netlify.com/drop (нужен бесплатный аккаунт).
2. Перетащите всю папку проекта в окно браузера.
3. Получите ссылку вида `https://имя.netlify.app` — её и раздаёте друзьям.
4. Обновление: перетащите папку ещё раз в разделе Deploys.

**Альтернативы:** Vercel (`vercel deploy`), GitHub Pages (Settings → Pages в репозитории), Cloudflare Pages. Всё бесплатно для такого объёма. Можно и на ваш VPS — просто положите файлы в папку веб-сервера (nginx/apache), приложению не нужен бэкенд.

> Важно: для установки на телефон (PWA) и работы камеры нужен HTTPS — все перечисленные хостинги дают его автоматически.

## Подключение Supabase (синхронизация между устройствами)

Без этого шага приложение работает в демо-режиме. С ним — у вас и друзей появятся аккаунты, и карточки будут общими для всех ваших устройств.

1. **Создайте проект**: зарегистрируйтесь на https://supabase.com (бесплатный тариф), нажмите **New project**, придумайте имя и пароль базы. Подождите пару минут, пока проект создастся.

2. **Создайте таблицы**: в меню слева откройте **SQL Editor**, вставьте содержимое файла `supabase_schema.sql` целиком и нажмите **Run**. Это создаст таблицы папок, карточек, настроек, права доступа (каждый видит только своё) и хранилище картинок.

3. **Возьмите ключи**: меню **Settings → API Keys**. Скопируйте:
   - **Project URL** (вида `https://xxxx.supabase.co`),
   - ключ **anon / public**.

4. **Впишите их в приложение**: откройте `js/config.js` и заполните:
   ```js
   window.KAR_CONFIG = {
     SUPABASE_URL: 'https://xxxx.supabase.co',
     SUPABASE_ANON_KEY: 'eyJ...ваш ключ...',
   };
   ```
   Ключ anon безопасно хранить в коде страницы — доступ к данным ограничивают правила на сервере.

5. **(Рекомендую для прототипа)** Отключите подтверждение почты, чтобы друзья входили сразу: **Authentication → Sign In / Providers → Email → выключите Confirm email**. Иначе после регистрации нужно кликнуть по ссылке из письма.

6. Загрузите обновлённую папку на хостинг заново. На экране входа появится форма почты и пароля.

Бесплатного тарифа Supabase (500 МБ базы, 1 ГБ файлов) для нескольких человек хватит с большим запасом — картинки автоматически сжимаются до 1024px перед загрузкой.

**Перенос карточек из демо-режима в облако:** в демо-режиме откройте Настройки → Экспорт → Скачать; войдите в аккаунт и там же сделайте Импорт этого файла. Картинки перенесутся в облако автоматически.

## Установка на телефон

Откройте сайт в браузере телефона:
- **Android / Chrome**: меню ⋮ → «Добавить на главный экран» (или «Установить приложение»).
- **iPhone / Safari**: кнопка «Поделиться» → «На экран “Домой”».

Приложение откроется без адресной строки, с иконкой вороны, и будет работать даже офлайн (в демо-режиме — полностью, в облачном — просмотр закэшированного).

## Управление с клавиатуры (на компьютере)

Во время повторения: **пробел / Enter** — перевернуть карточку, **← / →** или **1–2** — «не знаю» / «знаю», на телефоне — свайпы влево/вправо.

## Структура проекта

ES-модули, точка входа — `js/app.js` (подключён в `index.html`).

```
index.html              — оболочка PWA
css/style.css           — стили
css/fonts/              — локальные шрифты (Baloo 2, Nunito)
js/app.js               — запуск приложения
js/core/                — state, router
js/data/                — IndexedDB и Supabase stores
js/lib/                 — SRS, activity, stats, card-import, translate
js/ui/                  — UI-компоненты, shell, helpers
js/screens/             — экраны (auth, home, folder, review, settings, card-editor)
sw.js                   — service worker (офлайн-кэш)
manifest.webmanifest    — манифест PWA
icons/                  — иконки и иллюстрации
supabase_schema.sql     — схема базы для Supabase
```

Навигация по хэшу: `#home`, `#folder/:id`, `#review`, `#settings`. Новый JS-файл нужно добавить в `APP_FILES` в `sw.js`.

## Частые вопросы

**Данные в демо-режиме пропадут?** Они живут в IndexedDB браузера — переживают перезагрузки, но удалятся при очистке данных сайта. Делайте экспорт-бэкапы или подключите облако.

**Как добавить друга?** В облачном режиме он просто открывает вашу ссылку и нажимает «Создать аккаунт». Его коллекция полностью отдельная.

**Как поменять интервалы Лейтнера?** Настройки → выберите алгоритм «Лейтнер» → появятся поля интервалов для каждой коробки (в днях).

**У меня уже настроен Supabase — нужно ли что-то менять из-за поля «Описание»?** Да, один раз. Откройте свой проект на supabase.com → **SQL Editor** → вставьте и выполните:
```sql
alter table public.cards add column if not exists back_desc text default '';
```
Существующие карточки не пострадают — поле «Описание» у них будет просто пустым, а «Определение» (старый текст оборота) останется как было.
