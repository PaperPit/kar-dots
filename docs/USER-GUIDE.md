# Полное руководство: развернуть и пользоваться КАР-точками

Пошаговая инструкция **для любого пользователя** — без опыта программирования (где возможно).  
КАР-точки — open source: вы поднимаете **свой** экземпляр приложения и открываете его на компьютере и телефоне.

**Живое демо upstream:** [https://kar-tochki.pages.dev](https://kar-tochki.pages.dev)  
(можно попробовать сразу; для своих данных и друзей лучше свой деплой + свой Supabase.)

## 1. Какой способ выбрать

| Цель | Способ | Время |
|------|--------|-------|
| Попробовать без установки | [Демо](https://kar-tochki.pages.dev) | 1 мин |
| Свой сайт + YouTube/TTS API | [A — Cloudflare Pages](#2-способ-a-cloudflare-pages-рекомендуется) | ~20 мин |
| Только статика (без серверного YouTube) | [B — GitHub Pages](#3-способ-b-через-github-pages-только-статика) | ~15 мин |
| Только один компьютер | [C — локально](#4-способ-c-только-на-своём-компьютере) | ~3 мин |
| Карточки на телефоне **и** компе | A + [Supabase](#5-подключить-облако-supabase-sync--друзья) | +20 мин |
| Друзья с **отдельными** коллекциями | A + Supabase, раздать ссылку | +20 мин |

> **Важно:** установка на телефон как приложение и камера для картинок работают только по **HTTPS**-ссылке (Cloudflare Pages и т.п.). Просто «файл на диске» на телефон так не поставить.  
> Импорт из YouTube и Orpheus TTS нуждаются в **Pages Functions** — поэтому для полного функционала выбирайте способ A.

## 2. Способ A: Cloudflare Pages (рекомендуется)

Получите ссылку вида `https://ваше-имя.pages.dev` со статикой **и** API (`/api/yt-video`, `/api/yt-generate`, …).

Подробная шпаргалка для админа: **[cloudflare-pages-setup.md](./cloudflare-pages-setup.md)**.

### Вариант A1 — через GitHub (удобно обновлять)

1. Сделайте **Fork** https://github.com/PaperPit/kar-dots  
2. В Cloudflare: [dash.cloudflare.com](https://dash.cloudflare.com) → аккаунт → скопируйте **Account ID**  
3. Создайте [API Token](https://developers.cloudflare.com/fundamentals/api/get-started/create-token/) (шаблон **Edit Cloudflare Workers**)  
4. В форке GitHub → **Settings → Secrets and variables → Actions** добавьте:
   - `CLOUDFLARE_API_TOKEN`
   - `CLOUDFLARE_ACCOUNT_ID`
   - `SUPABASE_URL` и `SUPABASE_ANON_KEY` (если уже есть проект Supabase; иначе можно позже)
5. Один раз создайте проект Pages и KV (в терминале или по инструкции в [cloudflare-pages-setup.md](./cloudflare-pages-setup.md)):
   ```bash
   npx wrangler login
   npx wrangler pages project create ваше-имя --production-branch=main
   npx wrangler kv namespace create YT_JOBS
   ```
6. Запустите workflow **Deploy Cloudflare Pages** (Actions → Run workflow) или сделайте любой `git push` в `main`  
7. Откройте `https://ваше-имя.pages.dev` (имя проекта = имя в `wrangler pages project create`)

Секреты для YouTube/LLM на сервере: `wrangler pages secret put GEMINI_API_KEY` / `GROQ_API_KEY` / `SUPADATA_API_KEY` — см. [youtube-import-setup.md](./youtube-import-setup.md).

### Вариант A2 — ручной деплой из терминала

```bash
git clone https://github.com/PaperPit/kar-dots.git
cd kar-dots
npm install
npx wrangler login
npx wrangler pages project create kar-tochki --production-branch=main
npm run pages:deploy
```

В конце wrangler покажет URL. Обновление: снова `npm run pages:deploy` или push в `main` (если настроен Action).

### Проверка

1. Откройте ссылку в Chrome или Safari  
2. Экран входа с вороной и **«Попробовать без регистрации»**  
3. Приложение открылось — деплой успешен

## 3. Способ B: через GitHub Pages (только статика)

Подходит, если нужен простой HTTPS без серверных функций. **Импорт YouTube / серверный TTS на GitHub Pages не работают** (нет `/api/*`). Для полного функционала — [способ A](#2-способ-a-cloudflare-pages-рекомендуется).

### Шаг 1. Fork

1. На https://github.com/PaperPit/kar-dots нажмите **Fork** → **Create fork**

### Шаг 2. (Опционально) Supabase

В **своём** форке создайте файл `js/config.js` (через веб-редактор GitHub или локально):

1. Откройте `js/config.example.js` → скопируйте содержимое  
2. Создайте `js/config.js` с тем же содержимым  
3. Вставьте URL и anon key из Supabase ([раздел 5](#5-подключить-облако-supabase-sync--друзья))

> Файл `config.js` в оригинальном репозитории не коммитится. В **вашем** форке вы можете его добавить — это **ваши** ключи.

### Шаг 3. Включить Pages

1. В форке: **Settings** → **Pages**  
2. Source: **Deploy from a branch**  
3. Branch: **main**, folder: **/ (root)** → **Save**  
4. URL: `https://ВАШ-ЛОГИН.github.io/kar-dots/`

> Для продакшена с бандлом лучше Cloudflare (`dist/`). GitHub Pages из корня — упрощённый вариант для демо UI.

### Шаг 4. Открыть приложение

Перейдите по URL из Pages. Закладка — ваш постоянный адрес.

## 4. Способ C: только на своём компьютере

Подходит для пробы без интернета (после первой загрузки часть функций работает офлайн).

### Почему не «двойной клик по index.html»

Приложение использует современные модули JavaScript. В Chrome и Safari открытие `index.html` с диска (`file://`) **часто не работает**. Нужен маленький локальный «сервер».

### Windows / macOS / Linux — через Python

1. Скачайте и распакуйте проект (ZIP с GitHub: **Code → Download ZIP**)  
2. Откройте **Терминал** (macOS) или **cmd/PowerShell** (Windows)  
3. Перейдите в папку проекта, например:

```bash
cd ~/Downloads/kar-dots
```

4. Запустите сервер:

```bash
python3 -m http.server 8080
```

(На Windows может быть `python` вместо `python3`.)

5. В браузере откройте: **http://localhost:8080**  
6. Чтобы остановить сервер — `Ctrl+C` в терминале

### Альтернатива — Node.js (для разработчиков)

```bash
cd kar-dots
npm install
npm run dev
```

Откроется http://localhost:8080 с hot reload.

## 5. Подключить облако Supabase (sync + друзья)

Supabase — бесплатный backend **вашего** инстанса: регистрация, sync между устройствами, хранение картинок. Каждый пользователь видит **только свои** карточки.

### Шаг 1. Создать проект Supabase

1. https://supabase.com → **Start your project** → войти через GitHub  
2. **New project** → имя (например `kar-dots-home`), пароль БД, регион  
3. Подождите 1–2 минуты, пока проект создастся

### Шаг 2. Создать таблицы

1. В Supabase: слева **SQL Editor** → **New query**  
2. Откройте в скачанном проекте файл `supabase_schema.sql` (или по очереди файлы из `supabase/migrations/`)  
3. Скопируйте **весь** текст в SQL Editor → **Run**  
4. Должно быть сообщение об успехе, без красных ошибок

### Шаг 3. Взять ключи

1. **Settings** (шестерёнка) → **API**  
2. Скопируйте:
   - **Project URL** — `https://xxxxx.supabase.co`
   - **anon public** key (Publishable key)

### Шаг 4. Прописать ключи в приложении

1. В папке проекта скопируйте файл:

```
js/config.example.js  →  js/config.js
```

2. Откройте `js/config.js` в блокноте и вставьте ключи:

```js
export default {
  SUPABASE_URL: 'https://xxxxx.supabase.co',
  SUPABASE_ANON_KEY: 'eyJ...ваш ключ...',
};
```

3. Сохраните файл  
4. **Заново** задеплойте на Cloudflare (`npm run pages:deploy` или push в `main`) / обновите GitHub Pages

### Шаг 5. Упростить регистрацию (рекомендуется для друзей)

В Supabase: **Authentication** → **Providers** → **Email** → выключите **Confirm email**.  
Тогда после «Создать аккаунт» можно сразу входить без письма.

### Шаг 6. Проверить

1. Откройте ваш сайт  
2. Должны появиться поля **Почта** и **Пароль**  
3. **Создать аккаунт** → войти → главный экран с папками

Подробнее для админов: [DEPLOY.md](./DEPLOY.md).

## 6. Первый запуск приложения

После открытия сайта вы увидите экран с вороной.

### Режим «Попробовать без регистрации»

- Нажмите **«Попробовать без регистрации»**  
- Данные хранятся **только в этом браузере** (IndexedDB)  
- Между телефоном и компьютером **не синхронизируются**  
- Подходит для быстрой пробы

### Режим с аккаунтом (Supabase)

- Введите почту и пароль (минимум 6 символов)  
- **Создать аккаунт** — первый раз  
- **Войти** — следующие разы  
- Одна учётная запись = одна коллекция на **всех** устройствах с этим сайтом

### Что дальше

Создаётся первая папка с примером карточки. Дальше — [раздел 7](#7-как-пользоваться-на-компьютере).

## 7. Как пользоваться на компьютере

### Навигация

Вверху три раздела:

| Вкладка | Назначение |
|---------|------------|
| **Папки** | коробки, папки, список карточек, добавление |
| **Повторение** | учёба по SRS (счётчик «к повторению») |
| **Настройки** | алгоритм, лимиты, экспорт, голоса, интеграции |

### Создать папку и карточки

1. **Папки** → в коробке или на главной → **+ Новая папка**  
2. Откройте папку → **+ Добавить карточку**  
3. Заполните лицо и оборот; можно картинку, описание, жирный текст, ссылку  
4. **+ Добавить списком** — много карточек формата `слово — перевод`  
5. **Карточки из YouTube** — импорт по субтитрам (нужны API-ключи в Настройках)

### Повторение

1. **Повторение** (или кнопка **Повторить** на главной)  
2. Выберите режим: классика, ввод, пропуски, голос, пары, микс  
3. В классическом режиме:
   - **Пробел** или **Enter** или клик — перевернуть карточку  
   - **←** / **→** или **1** / **2** — «не знаю» / «знаю»  
4. В **Настройках → Интервальное повторение** выберите алгоритм: SM-2, FSRS или Лейтнер

### Установить как приложение на компьютер (PWA)

Нужна **HTTPS**-ссылка вашего деплоя (не `localhost` для закрепления у некоторых браузеров — localhost тоже часто работает).

**Google Chrome / Microsoft Edge (Windows, macOS):**

1. Откройте ваш сайт  
2. В адресной строке справа — иконка **Установить** (⊕ или монитор со стрелкой)  
3. **Установить** → приложение появится в меню Пуск / Launchpad / списке программ  
4. Открывается **без адресной строки**, как отдельное окно

**Safari (macOS):**

Safari хуже поддерживает установку PWA. Удобнее Chrome/Edge или добавление в Dock через **Файл → Добавить в Dock** (macOS Sonoma+).

### Горячие клавиши (повторение)

| Клавиша | Действие |
|---------|----------|
| Пробел, Enter | Перевернуть карточку |
| ← , 1 | Не знаю |
| → , 2 | Знаю |

## 8. Как установить на телефон (PWA)

Нужна **ваша HTTPS-ссылка** (Cloudflare Pages и т.д.).

### Android (Chrome)

1. Откройте ссылку в **Chrome**  
2. Войдите или «без регистрации»  
3. Меню **⋮** (три точки)  
4. **Добавить на главный экран** или **Установить приложение**  
5. Подтвердите — иконка вороны появится на рабочем столе  
6. Запуск **без адресной строки**, как обычное приложение

### iPhone / iPad (Safari)

1. Откройте ссылку **только в Safari** (не во встроенном браузере Telegram/VK)  
2. Кнопка **Поделиться** (квадрат со стрелкой вверх)  
3. Прокрутите → **На экран «Домой»**  
4. **Добавить**  
5. Иконка «КАР-точки» на домашнем экране

> На iOS PWA хранит данные отдельно от Safari. Если установили с главного экрана — всегда открывайте **с иконки**, иначе будут две разные «копии» данных.

### Свайпы на телефоне (классическое повторение)

После переворота карточки:

- **Свайп влево** — не знаю  
- **Свайп вправо** — знаю  

### Разрешения

- **Микрофон** — для режима «Голос» (Safari/Chrome спросят при первом использовании)  
- **Камера / галерея** — при добавлении картинки на карточку

### Офлайн на телефоне

- **Без регистрации:** после первого открытия онлайн многие экраны работа офлайн (кэш PWA)  
- **С аккаунтом:** просмотр закэшированного возможен; новые sync нужен интернет

## 9. Синхронизация: компьютер + телефон

Чтобы **одни и те же карточки** были на ПК и телефоне:

1. Разверните сайт с настроенным [Supabase](#5-подключить-облако-supabase-sync--друзья)  
2. На **компьютере**: создайте аккаунт, добавьте карточки  
3. На **телефоне**: откройте **ту же ссылку**, **Войти** с той же почтой и паролем  
4. Установите PWA с главного экрана (не обязательно, но удобнее)

Данные подтянутся из облака. Изменения на телефоне появятся на компьютере после синхронизации (нужен интернет).

### Перенос из «без регистрации» в аккаунт

1. В демо-режиме: **Настройки → Экспорт → Скачать JSON**  
2. Настройте Supabase и войдите в аккаунт  
3. **Настройки → Импорт** — выберите файл  
4. Картинки загрузятся в ваш Supabase Storage

## 10. Поделиться с друзьями

1. Разверните свой инстанс (способ A или B)  
2. Подключите [Supabase](#5-подключить-облако-supabase-sync--друзья)  
3. Отправьте друзьям ссылку, например `https://kar-tochki.pages.dev`  
4. Каждый нажимает **Создать аккаунт** со **своей** почтой  

**У каждого своя коллекция** — вы не видите чужие карточки, друзья не видят ваши.

Вы отвечаете только за хостинг (Cloudflare Pages бесплатно) и лимиты **вашего** Supabase (бесплатного тарифа обычно хватает на небольшую компанию).

## 11. Резервная копия

Регулярно делайте экспорт — даже в облачном режиме:

**Настройки → Экспорт → Скачать JSON**

Файл содержит папки, карточки, настройки. Храните в облаке (Google Drive, iCloud) или на диске.

Импорт того же файла восстанавливает данные или переносит их на новый инстанс.

## 12. Частые вопросы

**На телефоне не предлагает «Установить»**  
Убедитесь, что открыта **HTTPS**-ссылка деплоя, не локальный файл. На iPhone — только Safari.

**После очистки данных браузера всё пропало (демо-режим)**  
Демо хранится в браузере. Восстановление только из JSON-бэкапа, если делали экспорт.

**«Облачный режим не настроен»**  
Нет `js/config.js` с ключами Supabase. Создайте файл и перезалейте сайт.

**Друг не может зарегистрироваться — просит подтвердить почту**  
В Supabase отключите **Confirm email** (см. [шаг 5](#шаг-5-упростить-регистрацию-рекомендуется-для-друзей)).

**YouTube-импорт не работает**  
Нужны ключи Supadata / Gemini / Groq в **Настройках → Интеграции** и Cloudflare Pages Functions при деплое — см. [youtube-import-setup.md](./youtube-import-setup.md).

**Хочу нативное приложение в App Store**  
Для продвинутых: сборка через Xcode — [IOS.md](./IOS.md) (нужен Mac).
