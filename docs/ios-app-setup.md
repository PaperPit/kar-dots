# КАР-точки на iPhone через Xcode — полноценное приложение с авто-обновлениями

Эта инструкция превращает веб-приложение в настоящую иконку на домашнем экране iPhone (не ярлык браузера, а нативное приложение), и настраивает всё так, чтобы новые версии прилетали **сами**, без пересборки в Xcode при каждом изменении.

Проект уже подготовлен: в нём есть Capacitor (обёртка «веб → нативное приложение»), готовый Xcode-проект в папке `ios/`, идентификатор приложения `ru.kartochki.app` и имя «КАР-точки». Так что с нуля ничего создавать не нужно — только настроить подпись и один раз собрать.

---

## Главная идея автоматизации (прочитай первым)

У любого Capacitor-приложения есть два способа брать веб-контент:

1. **Вшитый пакет** (как сейчас): HTML/CSS/JS лежат внутри приложения. Любое изменение → надо заново копировать файлы и пересобирать в Xcode. Медленно и вручную.
2. **Живой сайт через `server.url`** (рекомендуется): приложение — это тонкая обёртка, которая открывает твой уже задеплоенный сайт. Тогда цикл обновления такой:

   ```
   правишь код → git push → Netlify/Vercel сам пересобирает сайт →
   → открываешь приложение на iPhone → видишь новую версию
   ```

   **Xcode для обновления контента больше не нужен вообще.** Это и есть «максимально автоматизировано». Бонусом серверные функции (импорт из YouTube, `/api/*`), Supabase-вход и офлайн-кэш (service worker) работают ровно как в браузере, потому что приложение грузится с настоящего адреса сайта.

Поэтому инструкция ниже сначала настраивает `server.url`, а уже потом собирает приложение. Вшитый пакет остаётся запасным вариантом (раздел в конце) — на случай, если хочешь полностью офлайновую сборку.

---

## Что понадобится

- **Mac с Xcode.** Установи Xcode из App Store (бесплатно, ~7–15 ГБ). После установки один раз открой его, чтобы он доустановил компоненты, и прими лицензию.
- **Apple ID.** Годится обычный бесплатный — но с важным ограничением (см. ниже про «7 дней»). Платный Apple Developer ($99/год) снимает ограничение и открывает TestFlight.
- **iPhone + кабель** (первый раз подключение по кабелю надёжнее, потом можно по Wi-Fi).
- **Твой задеплоенный адрес сайта** — тот, по которому приложение открывается в браузере и работает импорт из YouTube (например `https://твоё-имя.netlify.app`). Его видно в дашборде Netlify/Vercel. Он понадобится в шаге 1.

### Бесплатный Apple ID против платного — важно

| | Бесплатный Apple ID | Apple Developer ($99/год) |
|---|---|---|
| Установка на свой iPhone | да | да |
| Срок жизни приложения | **7 дней**, потом перестаёт открываться, пока снова не запустишь из Xcode | **1 год** |
| Сколько приложений | до 3 | много |
| TestFlight (установка «по воздуху», без кабеля) | нет | да |

Для личного использования бесплатного хватает, но раз в неделю придётся пересобирать из Xcode (30 секунд: подключил, нажал ▶). Если это раздражает — $99/год решают проблему и добавляют TestFlight (см. раздел «Максимальная автоматизация»).

## Шаг 1. Переключить приложение на живой сайт

Открой файл `capacitor.config.json` в корне проекта. Сейчас там:

```json
{
  "appId": "ru.kartochki.app",
  "appName": "КАР-точки",
  "webDir": "www",
  "ios": {
    "contentInset": "automatic",
    "allowsLinkPreview": false
  },
  "server": {
    "androidScheme": "https"
  }
}
```

Добавь в блок `server` строку `url` со **своим** адресом (замени на свой!):

```json
{
  "appId": "ru.kartochki.app",
  "appName": "КАР-точки",
  "webDir": "www",
  "ios": {
    "contentInset": "automatic",
    "allowsLinkPreview": false
  },
  "server": {
    "url": "https://твоё-имя.netlify.app",
    "androidScheme": "https"
  }
}
```

Требования к адресу: обязательно `https://`, без слэша в конце. Это должен быть тот же адрес, где работают серверные функции (импорт из YouTube), иначе в приложении импорт отвалится.

Теперь установи Capacitor CLI (один раз) и «протолкни» конфиг в iOS-проект:

```bash
cd ~/Claude/Projects/"Веб приложение Карточки"
npm install --save-dev @capacitor/cli
npx cap copy ios
```

`npx cap copy ios` копирует текущий `capacitor.config.json` и папку `www` внутрь Xcode-проекта. Эту команду нужно повторять только когда меняешь сам конфиг (адрес, имя, иконки) — для обычных правок сайта она не нужна, потому что контент грузится с живого адреса.

> Если увидишь предупреждение про «cleartext» или App Transport Security — оно не про тебя: адрес по `https`, всё в порядке.

## Шаг 2. Открыть проект в Xcode и настроить подпись

1. Открой Xcode-проект. В Finder зайди в папку проекта → `ios` → `App` и открой **`App.xcodeproj`** двойным кликом (именно `.xcodeproj`; этот проект использует Swift Package Manager, отдельный `.xcworkspace` не нужен).
2. В левой панели кликни по синей иконке проекта **App** (самый верх), затем в центральной панели выбери таргет **App** и вкладку **Signing & Capabilities**.
3. Поставь галочку **Automatically manage signing**.
4. В поле **Team** выбери свой Apple ID. Если его нет — нажми **Add an Account…**, войди под своим Apple ID, вернись и выбери его как Team (это «Personal Team»).
5. **Bundle Identifier**: сейчас `ru.kartochki.app`. Если Xcode подчеркнёт его красным с ошибкой вроде «bundle identifier is not available» — значит такой ID кто-то уже занял в системе Apple. Поменяй на свой уникальный, например `ru.<твоё-имя>.kartochki` (только латиница, цифры и точки). Запиши его — это постоянный идентификатор твоего приложения.

Если под полем Team нет красных ошибок — подпись настроена.

---

## Шаг 3. Подключить iPhone и запустить

1. Подключи iPhone к Mac кабелем. На телефоне появится запрос «Доверять этому компьютеру?» — нажми **Доверять** и введи код-пароль.
2. Включи на iPhone **режим разработчика**: Настройки → Конфиденциальность и безопасность → Режим разработчика → включить → перезагрузить телефон (это нужно один раз, на iOS 16+).
3. В Xcode вверху, рядом с кнопкой ▶, выбери устройство: нажми на выпадающий список целей и выбери свой iPhone по имени (не симулятор).
4. Нажми **▶ (Run)** или `Cmd + R`. Xcode соберёт приложение и установит его на телефон. Первая сборка идёт пару минут.
5. **Первый запуск упрётся в «Untrusted Developer».** Это нормально для личной подписи. На iPhone: Настройки → Основные → VPN и управление устройством → под своим Apple ID нажми **Доверять**. После этого запусти приложение с домашнего экрана.

Готово — на домашнем экране появилась иконка «КАР-точки», открывается как обычное приложение на весь экран, без адресной строки браузера.

## Как теперь выкатывать новые версии

Благодаря `server.url` повседневный цикл максимально короткий:

```bash
# поправил код
git add -A && git commit -m "описание" && git push
```

Netlify/Vercel сам пересоберёт сайт за минуту. На iPhone **полностью закрой приложение** (смахни его из списка запущенных) и открой заново — подтянется новая версия. Xcode при этом не нужен.

Пара нюансов:
- **Кэш service worker.** У приложения свой офлайн-кэш. Чтобы он гарантированно обновлялся, при каждом заметном релизе поднимай версию в `sw.js` (строка `const VERSION = 'kar-vXX'`) — старый кэш тогда сбрасывается автоматически. Ты это и так делаешь.
- **Первый запуск требует интернета** (приложение должно один раз загрузить сайт). Дальше работает офлайн из кэша.
- Раз в 7 дней (на бесплатном Apple ID) — пересобрать из Xcode кнопкой ▶, иначе iOS перестанет запускать приложение. Это НЕ про контент, а про саму подпись.

## Максимальная автоматизация (платный аккаунт): TestFlight + авто-сборка

Если оформишь Apple Developer ($99/год), можно убрать даже редкую возню с кабелем и «7 днями».

### Вариант A — TestFlight вручную (просто)
1. В Xcode: меню **Product → Archive**. Когда архив соберётся, откроется Organizer.
2. **Distribute App → App Store Connect → Upload**. Сборка уедет в TestFlight.
3. На iPhone установи приложение **TestFlight** из App Store и прими приглашение (себе на свой же Apple ID). Дальше приложение ставится и обновляется «по воздуху», без Mac и кабеля, и живёт год.

После этого контент всё равно обновляется мгновенно через `server.url` — TestFlight нужен только чтобы иногда обновлять саму нативную оболочку (иконку, разрешения, версию iOS).

### Вариант B — полная автоматизация через GitHub Actions + Fastlane (для энтузиастов)
Можно настроить так, что **каждый `git push` сам собирает нативное приложение и заливает его в TestFlight** без участия Mac. Кратко, что для этого нужно:

1. Установить [Fastlane](https://fastlane.tools) и сгенерировать в App Store Connect **API-ключ** (App Store Connect → Users and Access → Integrations → App Store Connect API).
2. Добавить в репозиторий `fastlane/Fastfile` с lane, который делает `build_app` + `upload_to_testflight`.
3. Создать GitHub Action на macOS-раннере (`runs-on: macos-14`), который на push запускает Fastlane; секреты (API-ключ, сертификаты) хранить в GitHub Secrets, сертификаты — через `fastlane match`.

Это мощно, но требует ощутимой разовой настройки. Для одного пользователя обычно избыточно: связка **`server.url` (контент сам) + TestFlight Вариант A (оболочка изредка)** уже даёт почти всё «по воздуху».

## Запасной путь: полностью вшитый пакет (офлайн-сборка без сайта)

Если по каким-то причинам не хочешь зависеть от живого адреса и предпочитаешь, чтобы весь код лежал внутри приложения:

1. Убери строку `"url": "..."` из `server` в `capacitor.config.json`.
2. Держи папку `www` в актуальном состоянии (это копия корня сайта: `index.html`, `css/`, `js/`, `sw.js`, `packs/`, `icons/`, `manifest.webmanifest`). Если правил сайт — синхронизируй `www` и проталкивай в iOS:
   ```bash
   npx cap copy ios
   ```
3. Пересобери в Xcode кнопкой ▶.

Минус очевиден: каждое обновление = ручной `cap copy` + пересборка. Плюс — работает без интернета с первого запуска и не зависит от хостинга. Но `/api/*` (импорт из YouTube) при таком режиме нужно вызывать по абсолютному адресу сайта, иначе серверные функции недоступны — ещё одна причина, почему для тебя удобнее `server.url`.

## Если что-то пошло не так

- **Белый экран при запуске** — приложение не смогло открыть `server.url`. Проверь: адрес указан с `https://`, сайт реально открывается в Safari на телефоне, есть интернет. Пере-протолкни конфиг: `npx cap copy ios` и пересобери.
- **«Untrusted Developer» и приложение не открывается** — Настройки → Основные → VPN и управление устройством → Доверять своему Apple ID.
- **Красная ошибка Bundle Identifier / Signing** — поменяй Bundle Identifier на уникальный (`ru.твоё-имя.kartochki`), убедись что выбран Team.
- **iPhone не виден в списке устройств** — переподключи кабель, разблокируй телефон, нажми «Доверять», включи режим разработчика (Настройки → Конфиденциальность и безопасность → Режим разработчика).
- **Приложение перестало открываться через неделю** — это лимит бесплатного Apple ID. Подключи iPhone, нажми в Xcode ▶ — снова заработает на 7 дней. Или оформи платный аккаунт.
- **Импорт из YouTube не работает в приложении, хотя в браузере работал** — значит `server.url` указывает не на тот адрес, где живут серверные функции. Впиши в `url` именно адрес с работающими `/api/*` и повтори `npx cap copy ios`.

## Короткая шпаргалка

```bash
# разовая настройка окружения
npm install --save-dev @capacitor/cli

# после правки capacitor.config.json (адрес/имя/иконки)
npx cap copy ios          # протолкнуть конфиг+www в Xcode-проект

# открыть проект в Xcode
open ios/App/App.xcodeproj

# обычный релиз контента (Xcode НЕ нужен)
git add -A && git commit -m "…" && git push
# затем на iPhone: закрыть и открыть приложение заново
```

Разовая настройка (шаги 1–3) занимает 15–20 минут. Дальше выкладка новой версии — это один `git push`.
