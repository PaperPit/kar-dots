# Импорт карточек из YouTube — настройка

Фича: в любой обычной папке появилась кнопка **«Карточки из YouTube»**. Вставляешь ссылку на ролик до 20 минут, выбираешь «Слова / Фразы / Слова + фразы» — приложение достаёт транскрипт, выделяет лексику с переводом на русский, отбрасывает то, что уже есть во встроенных паках (A0/A1/A2/phrases) и в ранее импортированных YouTube-карточках, и показывает превью с галочками. Отмеченное становится карточками текущей папки; в описании карточки — уровень CEFR, часть речи и кликабельный таймкод на момент в видео.

**Никаких платных сторонних сервисов.** Транскрипт достаётся напрямую из внутреннего API YouTube, без Supadata и подобных посредников; генерация карточек — через Gemini с автоматическим бесплатным резервом на Groq. Обратная сторона этого выбора и как её понимать — см. раздел «Известные риски» ниже.

## Архитектура

```
Браузер (js/screens/folder/youtube-dialog.js)
   │  POST /api/yt-video {url}
   ▼
netlify/functions/yt-video.mjs
   │   1. GET youtube.com/watch?v=… → достаём свежий INNERTUBE_API_KEY со страницы
   │   2. POST youtubei/v1/player (под видом Android-клиента) → метаданные,
   │      список субтитров, ссылки на аудиодорожки
   │   3а. Субтитры есть → сразу отдаём {video, transcript}
   │   3b. Субтитров нет, но есть прямая (без цифровой подписи) аудиодорожка →
   │       запускаем фоновую функцию и отвечаем {pending:true, jobId, video}
   │   3в. Ни того, ни другого → понятная ошибка "транскрипт недоступен"
   ▼ (только в случае 3b)
netlify/functions/yt-transcribe-background.mjs (Background Function, до 15 мин)
   │   скачивает аудио → Groq Whisper (speech-to-text) → пишет результат
   │   в Netlify Blobs (jobsStore)
   ▼
Браузер: пока pending — раз в ~2с опрашивает GET /api/yt-video?jobId=…
   ▼
Браузер: POST /api/yt-generate {title, lang, mode, segments}
   ▼
netlify/functions/yt-generate.mjs ──► Gemini generateContent (JSON-ответ)
                                    └─► если Gemini отказал и настроен
                                        GROQ_API_KEY → Groq chat-completions
                                        (Llama 3.3 70B) как резерв
   ▼
Браузер: сверка с паками и YouTube-папками (js/lib/youtube-import.js) → превью → store.createCard
```

Ни один API-ключ в браузер не попадает — все живут в переменных окружения Netlify.

## Регистрация (обе бесплатные)

1. **Google AI Studio** — основной провайдер генерации карточек.
   - На <https://aistudio.google.com> создай API key (free tier).
   - Бесплатные лимиты Gemini Flash (порядка сотен запросов в день) для личного использования более чем достаточны: 1 ролик = 1 запрос.
2. **Groq** — резервный провайдер (карточки) и распознавание речи для роликов без субтитров.
   - Зарегистрируйся на <https://console.groq.com>, в разделе API Keys создай ключ.
   - Бесплатный тариф: Llama 3.3 70B — щедрый дневной лимит запросов; Whisper (speech-to-text) — 20 запросов/мин, 2000/день, файлы до 25 МБ. Для личного использования этого с большим запасом хватает и на резерв генерации, и на расшифровку роликов без субтитров.
   - Настоятельно рекомендуется настроить оба ключа: без `GROQ_API_KEY` пропадает и резерв на случай квоты/сбоя Gemini, и вообще любая расшифровка роликов без встроенных субтитров.

Транскрипт из самого YouTube никакой регистрации не требует — используется его собственный (неофициальный) внутренний API.

## Где задаются ключи: настройки приложения или Netlify

Ключи можно задать **двумя способами**, работающими одновременно:

1. **Личные ключи в приложении** — Настройки → «Карточки из YouTube». Каждый пользователь может вставить свои ключи Gemini и Groq; они хранятся в его настройках (синхронизируются вместе с остальными настройками аккаунта) и передаются серверным функциям при каждом импорте. **Личный ключ всегда приоритетнее серверного** — так каждый работает со своими бесплатными лимитами и не толкается в общей квоте.
2. **Общие серверные ключи в Netlify** (Site configuration → Environment variables) — используются, когда личный ключ не указан:

| Переменная | Обязательна | Значение |
|---|---|---|
| `GEMINI_API_KEY` | нет (если у пользователей есть личные ключи) | ключ из Google AI Studio |
| `GEMINI_MODEL` | нет | по умолчанию `gemini-flash-latest` (см. ниже) |
| `GROQ_API_KEY` | нет (если у пользователей есть личные ключи) | ключ из console.groq.com — резерв генерации + распознавание речи без субтитров |

Если ни личный, ни серверный ключ не заданы, функции возвращают понятную ошибку `config` с подсказкой открыть настройки. Ключи из payload проходят проверку формата (`^[A-Za-z0-9_-]{20,200}$`) — мусор молча игнорируется и используется серверный ключ.

После добавления переменных нужен redeploy сайта. Функции подхватываются из `netlify/functions/` автоматически (`[functions]` в `netlify.toml`). Для `netlify/functions/yt-video.mjs` и `netlify/functions/yt-generate.mjs` нужен `@netlify/blobs` из `package.json` — он ставится автоматически при `npm install` на шаге сборки Netlify.

### О названии модели Gemini (важно)

Google периодически закрывает старые модели для новых API-ключей ещё до официальной даты отключения — так уже случилось с `gemini-2.5-flash` в июле 2026. Чтобы фича не ломалась при каждой смене поколений моделей, по умолчанию используется алиас **`gemini-flash-latest`** — он сам указывает на актуальную flash-модель Google.

Если Gemini всё же вернёт ошибку вида *"model … is no longer available"* или *"model not found"*:

1. Зайди в [ai.google.dev/gemini-api/docs/models](https://ai.google.dev/gemini-api/docs/models) и посмотри актуальное имя модели.
2. Пропиши его явно в переменной окружения `GEMINI_MODEL` в Netlify и сделай redeploy — это надёжнее алиаса, если хочешь зафиксировать конкретную модель.
3. Пока не поправишь — генерация карточек продолжит работать через Groq-резерв (если настроен `GROQ_API_KEY`), просто чуть менее качественно, чем через Gemini.

## Локальная разработка

Самый простой способ — dev-сервер проекта (статика + все три функции, без Netlify CLI):

```bash
npm install    # один раз — подтянет @netlify/blobs
npm run dev    # http://localhost:8080
```

Ключи локально берутся из настроек приложения (Настройки → «Карточки из YouTube») или из `.env` в корне (шаблон — `.env.example`; сам `.env` в `.gitignore`). Вместо Netlify Blobs dev-сервер использует общий in-memory стор — функции переключаются на него сами.

Альтернатива — официальный `netlify dev` (`npm i -g netlify-cli`): он подтянет переменные окружения из привязанного сайта (`netlify link`) и даст изолированное локальное хранилище Blobs.

Обычный `python3 -m http.server` для проверки YouTube-импорта не подходит — он не выполняет функции и возвращает «Ошибка сервера (501)» на любой POST.

## Известные риски (честно, без прикрас)

Это самый важный раздел этого документа — прочитай перед тем, как полагаться на фичу в проде.

Ранее транскрипт получался через Supadata — платный, но стабильный посредник, который сам решал все проблемы ниже за свой счёт. Сейчас вместо него используется прямой, неофициальный доступ к внутреннему API YouTube — это бесплатно, но менее надёжно. **Специально по твоему решению никакого скрытого fallback на Supadata или другой платный сервис нет** — если новый путь откажет, фича просто вернёт понятную ошибку.

Два конкретных источника нестабильности:

1. **YouTube блокирует часть запросов с IP дата-центров** (AWS, GCP, Azure — а значит и Netlify Functions, которые работают на AWS Lambda). Это известная, широко задокументированная проблема всей экосистемы неофициальных YouTube-инструментов (например, `youtube-transcript-api` в своей документации прямо предупреждает об этом и рекомендует платные резидентные прокси как единственное надёжное решение). На практике это может означать, что часть запросов будет получать ошибку `yt-blocked` — YouTube отклонил обращение с сервера. Есть шанс, что при повторной попытке через минуту сработает (Google не блокирует поголовно и не всегда), но гарантии нет.
2. **Расшифровка роликов без субтитров зависит от того, отдаёт ли YouTube прямую (без цифровой подписи) ссылку на аудиодорожку.** YouTube всё активнее внедряет протокол SABR, который может закрывать эти прямые ссылки или требовать более сложной расшифровки подписи — а её мы сознательно не реализовывали (это отдельный источник постоянной поддержки: инструменты вроде NewPipe/PipePipe требуют частых обновлений именно из-за этого). Если для конкретного видео прямой ссылки нет, ролик без субтитров получит ошибку `transcript-unavailable`.

Из этого следует практический вывод: **этот новый код не протестирован против живого YouTube** (в отличие от прежней Supadata-версии) — в песочнице, где он писался, не было сетевого доступа ни к youtube.com, ни к npm-реестру для локальной проверки. Первое реальное использование после деплоя — это и есть первая проверка. Если что-то не сработает на конкретном ролике, самое полезное — посмотреть код ошибки в ответе (`yt-blocked`, `transcript-unavailable`, `stt-failed`, `audio-too-large` и т.д., см. таблицу ниже) и решить на месте: попробовать другое видео, повторить попытку позже, либо (если проблема окажется систематической) вернуться к обсуждению добавления платного прокси/резервного сервиса.

## Ограничения и поведение при ошибках

- Ролики длиннее 20 минут отклоняются сразу по метаданным, до попытки получить транскрипт.
- Видео без субтитров: сначала пробуем распознать речь через Groq Whisper (фоновая функция, клиент опрашивает статус до ~3 минут); если аудио скачать/распознать не удалось — понятная ошибка, без скрытых платных fallback'ов.
- Приватные/удалённые видео, исчерпанные квоты, некорректный JSON от LLM, устаревшее имя модели — во всех случаях диалог возвращает понятное сообщение и позволяет попробовать снова.

| Код ошибки | Где возникает | Смысл |
|---|---|---|
| `yt-blocked` | `yt-video` | YouTube отклонил запрос с сервера (см. «Известные риски») |
| `yt-unavailable` | `yt-video` | Не удалось связаться с YouTube (сеть/таймаут) |
| `not-found` | `yt-video` | Видео удалено, приватное или требует входа |
| `not-playable` | `yt-video` | Видео недоступно по другой причине (регион, возраст и т.п.) |
| `too-long` | `yt-video` | Длиннее 20 минут |
| `transcript-unavailable` | `yt-video` | Нет ни субтитров, ни прямой аудиодорожки для расшифровки |
| `audio-fetch-failed` | фоновая расшифровка | Не удалось скачать аудио с YouTube |
| `audio-too-large` | фоновая расшифровка | Аудио больше ~24 МБ (лимит бесплатного Whisper) |
| `stt-failed` / `stt-empty` | фоновая расшифровка | Groq не смог распознать речь / вернул пусто |
| `config` | `yt-video`, `yt-generate` | Нет ни личного (в настройках), ни серверного API-ключа |
| `quota` | `yt-video`, `yt-generate` | Исчерпана бесплатная квота Groq и/или Gemini |
| `llm-failed` / `llm-bad-json` | `yt-generate` | Ошибка LLM или невалидный JSON в ответе |
| `no-cards` | `yt-generate` | Модель не нашла подходящей лексики в ролике |

- Netlify free: синхронная функция `yt-video`/`yt-generate` ограничена ~10 с — укладываемся за счёт того, что тяжёлая часть (скачивание + распознавание аудио) вынесена в отдельную Background Function (лимит 15 мин). `yt-generate` дополнительно экономит время за счёт компактного транскрипта (≤28k символов) и отключённого «размышления» модели.
- Эндпоинты `/api/*` публичны (как и сайт). Квоту защищают лимиты самих сервисов; при желании можно добавить проверку Supabase-JWT в функциях.

## Как определяются «уже известные» слова

- Все четыре встроенных пака (`packs/*.json`) — целиком, независимо от того, установлены ли они.
- Карточки текущей папки — целиком.
- Карточки остальных обычных папок — только созданные импортом из YouTube (признак: таймкод-ссылка `youtube.com/watch?v=` в описании).
- Сравнение — по нормализованной базовой форме: `walking/walked/walks` считаются формами `walk` (см. `stemVariants` в `js/lib/youtube-import.js`).
- Если LLM вернул и фразу, и входящие в неё слова — остаётся только фраза.

## Файлы фичи

| Файл | Роль |
|---|---|
| `netlify/functions/yt-video.mjs` | метаданные + субтитры напрямую из YouTube (Innertube API), лимит 20 мин, запуск фоновой расшифровки, polling |
| `netlify/functions/yt-transcribe-background.mjs` | фоновая расшифровка речи через Groq Whisper для роликов без субтитров |
| `netlify/functions/yt-generate.mjs` | промпт + вызов Gemini с автоматическим резервом на Groq (Llama 3.3 70B), валидация JSON-ответа, восстановление точных таймкодов по исходным субтитрам |
| `js/lib/youtube-import.js` | чистые утилиты: парсинг ссылки, нормализация, сверка, description |
| `js/lib/youtube-import-settings.js` | личные API-ключи: чтение из настроек, добавление в тело запросов |
| `js/screens/settings/sections/integrations.js` | секция настроек «Карточки из YouTube»: поля Gemini/Groq ключей |
| `js/screens/folder/youtube-dialog.js` | диалог: форма → прогресс → превью → создание карточек |
| `css/screens/youtube-import.css` | стили диалога |
| `tests/youtube-import.test.js` | vitest-тесты утилит |
| `package.json` | зависимость `@netlify/blobs` (хранилище состояния фоновой расшифровки) |

Изменённые: `js/screens/folder/index.js` (кнопка), `js/ui/constants.js` (иконка), `css/style.css` (@import), `sw.js` (precache, версия `kar-v12.3`), `netlify.toml` (`[functions]`).
